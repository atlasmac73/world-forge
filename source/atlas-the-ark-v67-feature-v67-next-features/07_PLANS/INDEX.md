## Execution Sequencing Rule

These plans are ordered by dependency, not by interest: MVP (1) precedes Private Beta (2)
precedes Production SaaS (3) precedes Enterprise Scale (4). Development Factory (5), Memory OS
(6), and Agent Workforce (7) are force-multipliers that support the others — do not let work on
6 or 7 distract from or delay 1. Per `CLAUDE.md` §3 and §10, the MVP core loop and Genesis Cycle
gating take priority over building out Memory OS levels beyond what's been explicitly approved
(see `docs/ATLAS_CURRENT_PRIORITY.md`) or implementing new agents.

## Claude Instruction

Before starting work under any plan here, check `docs/ATLAS_CURRENT_PRIORITY.md` for the
current owner-set sequencing — it can reorder or pause a plan without rewriting the plan file
itself.
